### **市場缺口與痛點分析**

### **缺口 1：陀螺儀邊端 AI 的缺失**

目前市場兩極分化：

- **高端方案**（視覺 AI）：功能全面但需高端硬體 + 網路 + 隱私顧慮
- **低端方案**（手機錄影）：成本低但完全無法量化反饋

**市場缺乏**：

- ✗ 成本 \$20-50，內置 IMU 感應器的「智能手環/腕帶」
- ✗ 無需網路，完全本地邊端運算的姿勢分析
- ✗ 實時反饋，延遲 <150ms 的系統

### **缺口 2：乒乓球特化功能的缺失**

現有通用 app：

- 能識別手臂動作，但無法分辨「正手 vs 反手」「攻 vs 防」
- 無法量化「旋轉強度」「擊球速度」「發球合法性」

**市場缺乏**：

- ✗ 乒乓球動作分類器（決策樹/ML）
- ✗ 專項反饋（「膝蓋不夠低」→ 「陀螺儀 Z 軸不足」的映射）

### **缺口 3：中學生可製作性的缺失**

商業方案太複雜（神經網絡、雲端服務器）；DIY 方案無法量化。

**市場缺乏**：

- ✗ 適合 STEM 中學生的「可組裝 + 可理解」的完整方案
- ✗ 易於驗證和優化的小規模 ML 模型（決策樹/KNN，非深度學習）

## **第二部分：創新科技解決方案構思**

### **核心洞察：為什麼陀螺儀 > 視覺？**

在乒乓球訓練場景中：

| 維度 | 視覺系統(MediaPipe) | 陀螺儀系統(IMU) |
| --- | --- | --- |
| **照明依賴** | 高（需光線充足） | 無（完全獨立） |
| **網路需求** | 必須（雲端計算） | 無（邊端本地） |
| **延遲** | 300-500ms | <150ms |
| **隱私** | 需上傳視頻 | 完全本地 |
| **球拍追蹤** | 困難（小物體） | 簡單（安裝於柄） |
| **成本** | \$10/月 訂閱 + 硬體 | \$20-30 一次性 |
| **中學生可製性** | 低（複雜部署） | 高（開源元件） |

**結論**：陀螺儀方案在「場景適應性、成本、隱私、中學生可製性」上**全面超越視覺系統**。

### **三階段技術實現方案**

### **方案 A：純邊端方案（推薦用於 MVP）**

完全不依賴手機計算，所有 ML 運行在微控制器

**優勢**

- ✓ 超低延遲（<100ms）
- ✓ 完全離線可用
- ✓ 隱私安全
- ✓ 手機只做顯示 + 記錄

**劣勢**

- ✗ 微控制器內存有限（決策樹深度受限）
- ✗ 模型精度相對較低（70-80%）

**技術選擇**

- 決策樹（樹深 5-7）或 KNN（k=3）
- 儲存 8-12 個特徵向量

---

### **方案 B：邊端 + 手機混合方案（後續優化）**

邊端做初步分類，手機進行精細分析 + 深度學習

**優勢**

- ✓ 精度可達 85-90%
- ✓ 支援更複雜的動作分類
- ✓ 可進行跨球員對比

**劣勢**

- ✗ 需網路傳輸（不過本地 WiFi 即可）
- ✗ 延遲增加至 200-300ms

**技術選擇**

- 邊端：快速規則分類
- 手機：TensorFlow Lite 輕量 CNN（50-100KB）

---

### **方案 C：雲邊端三層方案（商業版）**

邊端 → 手機 → 遠程教練雲平台

**適用於**：未來商業化或學校聯網部署

## **第四部分：MVP（最小可行產品）核心功能定義**

### **MVP 定位**

- **目標用戶**：乒乓球初級-中級球員、校隊
- **核心需求**：「我想知道自己的擊球動作對不對」
- **價值承諾**：「3 秒內告訴你動作類型 + 質量評分」

## **第五部分：技術棧建議**

### **整體技術架構表**

| 層級 | 元件 | 選擇方案 | 原因 | 成本 |
| --- | --- | --- | --- | --- |
| **感應器層** | 6 軸 IMU | MPU-6050 或 MPU-9250 | 成熟穩定、I2C 標準、成本低 | \$2-4 |
| **微控制器** | 主控芯片 | ESP32（推薦）或 Arduino Nano | ESP32 有內置 BLE，無需額外模組；Arduino 則需 HC-05 | \$5-8 / \$3-5 |
| **通訊** | 無線傳輸 | 藍牙 BLE（ESP32 原生） 或 藍牙 5.0 模組 | BLE 低功耗，範圍 10-50m，延遲低 | 0（內置）/ \$3-5 |
| **電源** | 供電 | 3.7V 鋰聚合物電池 500mAh + TP4056 充電模組 | 可內置於球拍柄，續航 8-10 小時 | \$2-4 |
| **編程環境** | 嵌入式開發 | Arduino IDE（C/C++） + FreeRTOS（可選） | 開源、社群龐大、學習資源豐富 | 免費 |
| **ML 框架** | 決策樹/KNN | scikit-learn（訓練）+ 手工 C++ 代碼（部署） | 輕量級，樹深限制自動保證嵌入式可用 | 免費 |
| **手機 App** | 跨平台開發 | Flutter 或 React Native | Flutter 性能更好，React Native 生態更完整；選 Flutter | 免費 |
| **藍牙通訊** | 協議 | UART over BLE（簡單）或 BLE Characteristic GATT | UART 更易理解實現 | 免費 |
| **本地儲存** | 數據持久化 | SQLite（App 端）+ SPIFFS（ESP32 內部） | 跨平台、查詢靈活 | 免費 |
| **聲音反饋** | TTS/音頻 | Flutter 內建 `flutter_tts` + Google TTS | 無需預錄音頻，節省儲存 | 免費 |

### **推薦技術棧完整清單**

**硬體部分**

```
感應器:
  ├─ MPU-6050 GY-521 模組（I2C 介面）
  └─ 6 軸：3 軸陀螺儀 + 3 軸加速度計

微控制器:
  ├─ ESP32-DevKit-V1（推薦）
  │  └─ 內置 WiFi/BLE，針腳豐富
  └─ Arduino Nano + HC-05 藍牙模組（備選）

電源管理:
  ├─ 3.7V/500mAh 鋰聚合物電池
  ├─ TP4056 充電模組
  └─ AMS1117-3.3V 穩壓器

機械結構:
  ├─ 3D 列印外殼（Thingiverse 開源模型）
  └─ 魔術貼/雙面膠固定於球拍柄

```

**軟體部分**

```
嵌入式軟體（C/C++）:
  ├─ Arduino IDE 編程環境
  ├─ MPU6050 庫：MPU6050.h（I2Device）
  ├─ 信號處理：
  │  ├─ 低通濾波器（LPF）去噪
  │  ├─ 峰值檢測算法
  │  └─ 特徵提取（8 維特徵向量）
  ├─ ML 模型：
  │  ├─ 決策樹（max_depth=5）
  │  └─ 編譯為 C++ if-else 規則
  ├─ 藍牙通訊：
  │  ├─ BluetoothSerial（ESP32 內置）
  │  └─ 協議：JSON / CSV 格式
  └─ 程式流程：採樣 → 特徵提取 → 分類 → 發送

手機應用（Flutter）:
  ├─ flutter_blue_plus：藍牙連接 & 掃描
  ├─ flutter_tts：語音合成反饋
  ├─ sqflite：本地 SQLite 儲存
  ├─ fl_chart：簡易進度圖表
  ├─ vibration：震動反饋
  └─ UI：
     ├─ 連接頁面（BLE 配對）
     ├─ 訓練主頁（實時顯示分數）
     └─ 進度頁面（7 日統計）

模型訓練（Python + scikit-learn）:
  ├─ 數據：CSV 格式（時間序列傳感數據）
  ├─ 特徵工程：計算 8 個統計特徵
  ├─ 決策樹訓練：
  │  └─ DecisionTreeClassifier(max_depth=5)
  ├─ 驗證：Confusion Matrix、F1-Score
  └─ 導出：樹結構 → C++ if-else 規則

```
